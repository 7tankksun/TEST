Streamlit 생활정보 (날씨 + 에이전트)
====================================

로컬 실행
---------
  cd streamlit
  py -3 -m venv .venv
  .venv\Scripts\activate
  pip install -r requirements.txt
  streamlit run app.py

  브라우저: http://localhost:8509

Synology Container Manager — 실행 명령 (http.server 와 다름)
----------------------------------------------------------------
  정적 사이트: python3 -m http.server 8508 --bind 0.0.0.0 --directory /app
  Streamlit 은 아래처럼 streamlit run 사용 (작업 디렉터리 /app, app.py 기준):

  streamlit run app.py --server.address 0.0.0.0 --server.port 8509 --server.headless true --browser.gatherUsageStats false

  또는: python -m streamlit run app.py --server.address 0.0.0.0 --server.port 8509 --server.headless true --browser.gatherUsageStats false

  이 저장소의 Dockerfile 은 entrypoint.sh 로 위와 동일하게 띄움.
  DSM에서 컨테이너 만들 때: 실행 명령은 비워 두는 것이 안전함(덮어쓰기 충돌 방지).

python:latest 만 쓸 때 (이미지에 streamlit 없음 — 흔한 실패)
----------------------------------------------------------
  증상: exec: "streamlit": executable file not found in $PATH

  원인: 공식 python 이미지에는 pip 패키지 streamlit 이 설치되어 있지 않음.

  해결 A — 권장: 이 폴더 Dockerfile 로 이미지 빌드 후 그 이미지로 컨테이너 생성
    (python:3.11-slim + requirements.txt 이미 설치됨)

  해결 B — GUI만 빠르게: 이미지는 python:latest 유지, 실행 명령을 아래 한 줄로 교체
    (볼륨 /app 에 requirements.txt, app.py 있어야 함)

  sh -c "pip install --no-cache-dir -r /app/requirements.txt && python3 -m streamlit run /app/app.py --server.address 0.0.0.0 --server.port 8509 --server.headless true --server.enableCORS false --server.enableXsrfProtection false --browser.gatherUsageStats false"

  실행 명령에 streamlit 이라는 단어만 넣지 말 것 (PATH 에 없음). 반드시 python3 -m streamlit ...

실패할 때 (로그 먼저 확인)
--------------------------
  Container Manager → 해당 컨테이너 → 로그 에서 메시지 확인.

  - streamlit: not found → python3 -m streamlit ... 로 바꾸거나, 이 폴더 Dockerfile 로 빌드한 이미지 사용
  - No such file ... app.py → 작업 디렉터리 /app, app.py 위치·볼륨 마운트 확인
  - ModuleNotFoundError: streamlit → pip install -r requirements.txt (순수 python 이미지인 경우)
  - Address already in use → 포트 8509 충돌, 다른 컨테이너/서비스 중지 또는 포트 변경

로그에 "You can now view..." 가 보이는데 브라우저만 안 될 때
--------------------------------------------------------
  앱은 컨테이너 안에서 정상 기동한 상태다. 아래를 순서대로 본다.

  1) 포트 매핑 — 가장 흔함
     Container Manager → 해당 컨테이너 → 설정(또는 편집) → 포트
     로컬(호스트) 8509 → 컨테이너 8509, TCP.
     여기가 비어 있으면 NAS 밖(또는 DSM)에서 8509로 들어온 연결이 앱에 닿지 않는다.

  2) 주소 — 집 안에서는 공인 IP 말고 NAS 내부 IP
     같은 공유기 Wi-Fi/랜: http://192.168.x.x:8509
     로그의 External URL(예: 175.x.x.x)은 공유기에서 8509→NAS 로 포트포워딩이 있어야
     인터넷(밖)에서 통한다. 많은 공유기는 집 안에서 공인 IP로 접속(헤어핀 NAT)이 안 되므로,
     집에서는 반드시 192.168.x.x 를 쓴다.

  3) DSM 방화벽 — TCP 8509 허용 규칙이 있는지(제어 패널 → 보안 → 방화벽).

  4) 바인딩 — 실행 명령에 --server.address 0.0.0.0 이 빠지면 포트 매핑이 실패할 수 있다.
     SYNOLOGY_실행명령.txt 와 동일한지 확인. 이 프로젝트는 .streamlit/config.toml 로도 고정한다.


  수동 이미지에 넣을 때 권장 한 줄 (app.py 가 /app 에 있을 때):

  python3 -m streamlit run /app/app.py --server.address 0.0.0.0 --server.port 8509 --server.headless true --server.enableCORS false --server.enableXsrfProtection false --browser.gatherUsageStats false

NAS (Docker) — 공통
-------------------
  1) 이 streamlit 폴더를 NAS에 복사 (예: File Station /volume1/docker/streamlit)
  2) SSH 또는 터미널에서:
       cd /volume1/docker/streamlit
       docker compose -f docker-compose.nas.yml build
       docker compose -f docker-compose.nas.yml up -d
  3) 브라우저: http://NAS_내부IP:8509
     (접속 안 되면 제어 패널 → 보안 → 방화벽에서 TCP 8509 허용 규칙 추가)

Synology DSM에서 설정 (동일하게 하면 됨)
----------------------------------------
  A) Container Manager (권장)
     - 이미지 또는 프로젝트: "프로젝트" → 만들기 → streamlit 폴더의 docker-compose.nas.yml 지정
     - 또는 "컨테이너"에서 수동 생성 시:
         포트: 호스트 8509 → 컨테이너 8509
         환경변수(선택): OPENAI_API_KEY, OPENAI_MODEL
     - Synology Drive로 PC의 SynologyDrive/streamlit 를 NAS와 동기화해 두었다면,
       그 경로를 볼륨/빌드 경로로 지정해도 됨 (경로만 NAS 실제 경로로 맞출 것)

  B) 역프록시 + HTTPS (포털 https://life.도메인 링크용) — 안 되면 거의 여기
     - DNS: life.junna3d.i234.me 처럼 **life** 호스트가 NAS(또는 공유기) 공인 IP를 가리키는
       A 레코드(또는 DDNS 동일 규칙)가 있어야 함. 메인 도메인만 있고 life 가 없으면 404/연결 실패.
     - 제어 패널 → 로그인 포털 → 고급 → 역방향 프록시 → 규칙 편집
       · 소스: HTTPS, 호스트 이름 **life.본인도메인**, 포트 443
       · 대상: HTTP, **127.0.0.1** 또는 NAS 내부 IP, 포트 **8509**
     - **커스텀 헤더(필수에 가깝다)**: 같은 규칙에서「커스텀 헤더」→「생성」→
       드롭다운에 있으면 **WebSocket** 프리셋 적용.
       Streamlit 은 WebSocket 으로 화면을 붙이므로, 이게 없으면 https 로는 뜨지 않거나
       무한 로딩만 난다.
     - SSL 인증서: 제어 패널 → 보안 → 인증서 → **life.도메인** 이 포함된 인증서를
       해당 FQDN/역프록시에 쓰도록 설정(와일드카드 *.도메인 또는 life 별도 추가).
     - 포털( portal.py )은 기본으로 https://life.<도메인> 을 가리키므로,
       위 호스트 이름이 PORTAL_BASE_DOMAIN 과 맞아야 함.
     - IP:8509 만 쓸 때는 portal 컨테이너에 STREAMLIT_URL=http://NAS_IP:8509 설정
     - 이 저장소의 .streamlit/config.toml 은 역프록시 호환용으로 enableCORS/XSRF 등을 맞춤.
       NAS 에 반영 후 Streamlit 컨테이너 재시작.

  C) 포털과 같이 쓰기
     - portal/docker-compose.yml 의 environment 에 예:
         STREAMLIT_URL=https://life.본인도메인
       또는
         STREAMLIT_URL=http://192.168.x.x:8509
     - 포털 컨테이너 재시작 후 메인 페이지에「생활 정보 · 날씨」카드 링크 확인

  D) 날씨 API
     - Open-Meteo는 NAS에서 인터넷(HTTPS) 나가야 함.
     - 방화벽/프록시가 전부 차단하면 날씨 탭만 실패할 수 있음 (앱 자체는 뜸)

OpenAI 에이전트 (선택)
----------------------
  docker-compose 에서 환경변수:
    OPENAI_API_KEY=sk-...
    OPENAI_MODEL=gpt-4o-mini

  키가 없으면 규칙 기반 한국어 응답만 사용합니다.

날씨 데이터
-----------
  Open-Meteo 공개 API (키 불필요). NAS가 외부 HTTPS를 막으면 날씨 탭이 실패할 수 있습니다.

포털(portal.py) 연동
--------------------
  기본 링크: https://life.<PORTAL과 동일 도메인> (역프록시에서 8509로 넘기기)
  또는 portal/docker-compose environment 에:
    STREAMLIT_URL=http://NAS_IP:8509
