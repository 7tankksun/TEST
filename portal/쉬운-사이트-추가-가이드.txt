새 사이트 추가 — 쉬운 방법 (OPIC부터)
====================================

[지금 하신 방식]
- 사이트마다 "컨테이너 하나 + python 이미지 + 포트" → 사이트 늘어날수록 UI에서 반복 설계 (귀찮음, 맞음)

[앞으로 나누는 기준]
(1) 정적 웹 — HTML/JS/CSS/json 만 있으면 됨
    → OPIC 암기 사이트, 랜딩 페이지, 문서
    → Docker의 python이 필요 없음.

(2) Python으로 돌리는 앱
    → 지금 my-web-xxx 처럼, 그때만 “앱용 컨테이너”를 쓰면 됨.

────────────────────────────────
OPIC (정적) — 가장 쉬운 2가지
────────────────────────────────

A) Docker 안 쓰기 (권장, 설정 적음)
   1) PC에서 public 폴더를 NAS에 복사
      예: /volume1/web/opic/ (안에 index.html, app.js, styles.css, data/ …)
   2) DSM > 웹 스테이션(또는 Web Station) 에서
      - 가상 호스트 추가
      - 호스트 이름: opic.junna3d.i234.me
      - 문서 루트: 위 폴더
   3) 로그인 포털(역프록시) 쓰는 경우에만, HTTPS/인증서 맞추기
   → 새 "이미지/컨테이너" 없음, 포털(8500)과도 별개.

B) Docker는 쓰되 python 말고 nginx만
   - opic-study-site/public 만 NAS에 두고
   - 그 폴더 옆에 첨부한 docker-compose (nginx) 한 번만 실행
   - 포트 8508, 이미지는 nginx:alpine (가볍고, python보다 낫다)

────────────────────────────────
portal.py 와의 관계
────────────────────────────────
- 포털은 "링크 모음"일 뿐, OPIC이 Docker인지 Web Station인지는 상관 없음.
- opic.junna3d… 로 잘 뜨면, portal 에서 OPIC_URL 안 넣어도 기본 서브도메인으로 감.
- DDNS/역프록시가 그 호스트로만 연결되면 됨.

────────────────────────────────
7개 my-web 처럼 "앞으로도" 줄이는 팁
────────────────────────────────

  (지금 당장 필수는 아님, OPIC만 위 A/B로 빼는 것만으로도 부담이 크게 줄어듦)
